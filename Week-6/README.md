# Week 6: Git & GitHub Hands-on Exercises

This directory contains the Git & GitHub hands-on exercises completed during **Week 6** of the **Cognizant Digital Nurture Java FSE Deep Skilling Program**.

## Contents

- **GIT hands-on 1-5.docx** – A consolidated document containing the solutions, screenshots, objectives, and execution steps for all five Git hands-on exercises.

# Hands-on Summary

## 1. Foundational Git & GitHub Operations

**Objective**
- Learn the basic Git workflow by creating a local repository, configuring Git, tracking files, committing changes, and connecting to GitHub.

**Topics Covered**
- Git installation verification
- Git configuration
- Repository initialization
- File tracking
- Local commits
- Remote repository setup
- GitHub push

**Key Commands**
- `git --version`
- `git config`
- `git init`
- `git status`
- `git add`
- `git commit`
- `git remote add`
- `git push`

**Outcome**
- Successfully created a local Git repository and pushed the initial project to GitHub.

---

## 2. File Exclusion using `.gitignore`

**Objective**
- Learn how to prevent unnecessary files and folders from being tracked by Git.

**Topics Covered**
- Creating `.gitignore`
- Ignoring log files
- Ignoring directories

**Key Commands**
- `git status`
- `git add`
- `git commit`
- `git push`

**Outcome**
- Configured Git to ignore `.log` files and the `logs` directory while tracking only the required project files.

---

## 3. Branching and Merging

**Objective**
- Understand how Git branches enable independent development and how changes are merged into the main branch.

**Topics Covered**
- Branch creation
- Branch switching
- Feature development
- Branch merging
- Branch deletion

**Key Commands**
- `git branch`
- `git checkout`
- `git merge`
- `git log`
- `git branch -d`

**Outcome**
- Created a feature branch, committed new changes, merged them into the `main` branch, and removed the merged branch.

---

## 4. Merge Conflict Resolution

**Objective**
- Learn how merge conflicts occur and how they can be resolved manually.

**Topics Covered**
- Creating conflicting changes
- Merge conflict identification
- Conflict resolution
- Merge commit
- Updating `.gitignore`

**Key Commands**
- `git merge`
- `git add`
- `git commit`
- `git log`

**Outcome**
- Successfully resolved a merge conflict in `hello.xml`, committed the resolved changes, and synchronized the repository.

---

## 5. Remote Repository Synchronization

**Objective**
- Synchronize the local Git repository with the remote GitHub repository.

**Topics Covered**
- Repository verification
- Branch verification
- Pulling latest changes
- Pushing local commits

**Key Commands**
- `git status`
- `git branch -a`
- `git pull`
- `git push`

**Outcome**
- Verified that the local and remote repositories were fully synchronized and up to date.
